基本の大福店

子クラスは親クラスの全機能 + 新機能を持つ!

ステップ1: 親クラスを作る

# 親クラスの定義 class DaifukuShop: """大福店の基本クラス""" def __init__(self, owner_name, stock): # self = このインスタンス(このお店)自身 self.owner_name = owner_name # 店長名を保存 self.stock = stock # 在庫数を保存 self.sold = 0 # 販売数は0から print(f"🏪 {owner_name}店長の店を開店!") def sell(self, quantity): """大福を販売するメソッド""" if quantity > self.stock: print("❌ 在庫不足!") return False self.stock -= quantity # 在庫を減らす self.sold += quantity # 販売数を増やす print(f"💰 {quantity}個売れた!") return True

【一行ずつ解説】

class DaifukuShop:
→ 「DaifukuShop」という名前のクラス(設計図)を作る
→ class = クラスを作る命令
→ : (コロン) = ここからクラスの中身という印
def __init__(self, owner_name, stock):
→ __init__ = 初期化メソッド(お店を作る時に自動で実行される)
→ self = このお店自身
→ owner_name = 店長の名前(引数)
→ stock = 在庫の数(引数)
self.owner_name = owner_name
→ 引数でもらった店長名を、このお店の変数として保存
→ self.変数名 = インスタンス変数(このお店専用の変数)
def sell(self, quantity):
→ sell = 販売するメソッド(機能)
→ quantity = 売る個数(引数)
self.stock -= quantity
→ 在庫から売った分を引く
→ -= は「引き算して代入」の意味
→ self.stock = self.stock - quantity と同じ

ステップ2: 子クラスを作る (継承!)

class PremiumDaifukuShop(DaifukuShop): """プレミアム大福店 - 親を継承!""" def __init__(self, owner_name, stock, vip_count): # super() = 親クラスを呼び出す魔法の言葉 super().__init__(owner_name, stock) # 子クラス独自の変数 self.vip_count = vip_count print(f"✨ VIP会員: {vip_count}名") def sell_premium(self, quantity): """プレミアム販売 - 子クラス独自の機能!""" # 親のsellメソッドを使う result = super().sell(quantity) if result: price = quantity * 225 # 高級価格 print(f"✨ プレミアム価格: ¥{price}") return result

【継承の仕組み】

class PremiumDaifukuShop(DaifukuShop):
→ カッコの中に親クラス名を書く!
→ これで「DaifukuShopを継承します」という宣言
→ 親の機能が全部使えるようになる
super().__init__(owner_name, stock)
→ super() = 親クラスのこと
→ 親の__init__を呼ぶ = 親の初期化処理を実行
→ これで owner_name と stock が親と同じように設定される
→ super()を忘れると親の初期化がされないので注意!
self.vip_count = vip_count
→ 子クラス独自の変数を追加
→ 親にはない、プレミアム店だけの情報
→ 親の変数 + 子の変数 = 両方持てる
def sell_premium(self, quantity):
→ 子クラス独自のメソッド(機能)を追加
→ 親のsellに加えて、sell_premiumも使える!
result = super().sell(quantity)
→ 親のsellメソッドを呼んで使う
→ 継承してるから親のメソッドが使える!
→ 親の機能を再利用できるのが継承の利点

ステップ3: 使ってみる!

# 親クラスのインスタンスを作る basic = DaifukuShop("うさうさ", 20) basic.sell(5) # 子クラスのインスタンスを作る premium = PremiumDaifukuShop("もちもち", 30, 5) premium.sell(3) # 親から継承したメソッド premium.sell_premium(2) # 子クラス独自のメソッド

【使い方の解説】

basic = DaifukuShop("うさうさ", 20)
→ 親クラスのインスタンス(お店)を作る
→ "うさうさ"店長、在庫20個でスタート
→ クラス名() でインスタンスを作成
premium = PremiumDaifukuShop("もちもち", 30, 5)
→ 子クラスのインスタンス(お店)を作る
→ VIP会員5名の情報も追加
→ 引数3つ必要(owner_name, stock, vip_count)
premium.sell(3)
→ 子クラスだけど、親のメソッドが使える!
→ これが継承の力!
→ 親で定義したsellメソッドを呼び出せる
premium.sell_premium(2)
→ 子クラス独自のメソッドも使える
→ 親の機能 + 子の機能 = 両方使える!

✅ 単一継承のポイント

  • 一番シンプルで理解しやすい
  • 90%のケースはこれで十分
  • 親の機能を全部使える + 新機能も追加できる
  • 迷ったらまずこれを使う!
  • is-a関係が成り立つか確認 (プレミアム店「は」大福店「である」)

👨‍👩‍👧 階層継承 - 親→子→孫と続く継承

📖 イメージ図

祖父クラス (DaifukuShop)
基本店
↓
父クラス (PremiumDaifukuShop)
プレミアム店
↓
孫クラス (VIPDaifukuShop)
VIP専門店

段階的に機能を追加していく!

孫クラスを作る

class VIPDaifukuShop(PremiumDaifukuShop): """VIP専門店 - プレミアム店を継承!""" def __init__(self, owner_name, stock, vip_count, concierge): # 親(PremiumDaifukuShop)の__init__を呼ぶ # すると自動的に祖父(DaifukuShop)も初期化される! super().__init__(owner_name, stock, vip_count) # 孫クラス独自の変数 self.concierge = concierge print(f"👑 コンシェルジュ: {concierge}名") def sell_vip_exclusive(self, quantity): """VIP限定販売 - 孫クラス独自の機能!""" result = super().sell_premium(quantity) if result: price = quantity * 450 # 超高級 print(f"👑 VIP限定: ¥{price}") return result

【階層継承の仕組み】

class VIPDaifukuShop(PremiumDaifukuShop):
→ プレミアム店(父)を継承する
→ すると祖父の機能も自動的に受け継ぐ!
→ 継承の連鎖が起こる
super().__init__(owner_name, stock, vip_count)
→ 父のPremiumDaifukuShopの__init__を呼ぶ
→ 父の中でさらに祖父の__init__が呼ばれる
→ つまり: 孫 → 父 → 祖父 の順に初期化される
→ super()の連鎖が自動的に起こる
【継承の連鎖】
祖父: owner_name, stock, sell()
父: 祖父の全部 + vip_count, sell_premium()
孫: 父の全部 + concierge, sell_vip_exclusive()

→ 孫は3世代全部の機能が使える!

孫クラスを使ってみる

vip = VIPDaifukuShop("ぴょん", 40, 10, 3) vip.sell(2) # 祖父から継承 vip.sell_premium(3) # 父から継承 vip.sell_vip_exclusive(1) # 孫独自

【3世代の機能を全部使える】

vip.sell(2)
→ 祖父クラス(DaifukuShop)のメソッド
→ 孫でも使える!
vip.sell_premium(3)
→ 父クラス(PremiumDaifukuShop)のメソッド
→ 孫でも使える!
vip.sell_vip_exclusive(1)
→ 孫クラス独自のメソッド

つまり、孫は3世代全部の機能が使える!
祖父の機能 + 父の機能 + 孫の機能 = 全部!

⚠️ 注意点

  • 階層は2-3階層までがおすすめ
  • 深すぎると複雑になる (5階層以上はNG)
  • super()の連鎖を理解する
  • どのクラスから何を継承してるか把握する

4つの親クラスを作る

# 親A: 店舗機能 class ShopBase: def __init__(self, owner_name, stock): self.owner_name = owner_name self.stock = stock # 親B: オンライン機能 class OnlineServiceMixin: def __init__(self): self.online_orders = 0 # 親C: 配達機能 class DeliveryServiceMixin: def __init__(self): self.deliveries = 0 # 親D: 会員機能 class MembershipServiceMixin: def __init__(self): self.members = []

【4つの独立した機能】

親A (ShopBase):
→ 店舗としての基本機能
→ 店長名と在庫を管理
親B (OnlineServiceMixin):
→ オンライン注文の機能
→ Mixin = 機能追加専用クラス
→ 注文数を管理
親C (DeliveryServiceMixin):
→ 配達サービスの機能
→ 配達回数を管理
親D (MembershipServiceMixin):
→ 会員管理の機能
→ 会員リストを管理

それぞれ独立した機能 = 組み合わせ自由!

4つを統合した子クラス

class HybridDaifukuShop(ShopBase, OnlineServiceMixin, DeliveryServiceMixin, MembershipServiceMixin): """ハイブリッド店 - 4つの親を継承!""" def __init__(self, owner_name, stock): # 全ての親を初期化する必要がある! ShopBase.__init__(self, owner_name, stock) OnlineServiceMixin.__init__(self) DeliveryServiceMixin.__init__(self) MembershipServiceMixin.__init__(self) print("✅ 全機能OK!") def process_order(self, customer, quantity, address): """統合注文処理 - 全機能を組み合わせ!""" print(f"\n注文処理: {customer}様") # 1. オンライン注文(親B) self.online_orders += 1 print(f"🌐 注文 #{self.online_orders}") # 2. 販売(親A) if quantity <= self.stock: self.stock -= quantity # 3. 配達(親C) self.deliveries += 1 print(f"🚚 配達: {address}へ") # 4. 会員登録(親D) if customer not in self.members: self.members.append(customer) print(f"💳 会員登録: {customer}様") print("✅ 注文完了\n")

【多重継承の書き方】

class HybridDaifukuShop(親A, 親B, 親C, 親D):
→ カッコの中に複数の親をカンマ区切りで書く!
→ これで4つ全部の機能を継承
→ 順番は重要（左から優先される）
親A.__init__(self, owner_name, stock)
→ 親Aを直接呼んで初期化
→ 親クラス名.__init__() の形式
→ 注意: super()じゃなくて親名.__init__()を使う!
→ なぜなら複数の親があるから
全ての親を初期化する
→ 親A、B、C、Dを全部初期化
→ 一つでも忘れるとその機能が使えない
→ 多重継承の難しいところ
def process_order(...):
→ 4つの親の機能を組み合わせて使う
→ self.online_orders (親B)
→ self.stock (親A)
→ self.deliveries (親C)
→ self.members (親D)
→ 全部使える!

使ってみる

hybrid = HybridDaifukuShop("ふわふわ", 50) hybrid.process_order("田中太郎", 5, "東京都渋谷区") hybrid.process_order("佐藤花子", 3, "東京都新宿区") # 統計表示 print(f"在庫: {hybrid.stock}個") print(f"オンライン注文: {hybrid.online_orders}件") print(f"配達回数: {hybrid.deliveries}回") print(f"会員数: {len(hybrid.members)}名")

⚠️ 注意点

  • 一番複雑なパターン
  • 慎重に使う必要がある
  • 全ての親を初期化すること (忘れやすい!)
  • 本当に必要な時だけ使う
  • MRO (メソッド解決順序) を理解する必要がある
  • ダイヤモンド継承問題に注意
